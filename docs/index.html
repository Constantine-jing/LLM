Part 1: Foundations

What is a “Probe”?

A probe is a simple classifier (often linear) trained on the intermediate representations of a neural network to predict some property of interest.

Key idea: Neural networks learn rich internal representations — probes let us measure what information is encoded at each layer.

Analogy from Alain & Bengio (2016):

Think of probes as thermometers — they measure the temperature at different locations in the model without affecting the model itself.

Important

Critical rule: Probes must never influence the model’s training — gradients are blocked (via stop_gradient).

Alain & Bengio (2016): The Original Paper

“Understanding Intermediate Layers Using Linear Classifier Probes”

Core contributions:

  • Introduced probes as a general tool for understanding deep neural networks
  • Demonstrated probes on Inception v3 and ResNet-50 (ImageNet)
  • Discovered that linear separability increases monotonically through layers

The setup:

At each layer \(k\), take features \(h_k\) and fit a linear classifier:

\[f_k: h_k \mapsto \text{softmax}(W h_k + b)\]

Minimize cross-entropy loss. The probe parameters \((W, b)\) are trained independently from the model.

Why linear?

  • Training a linear classifier with softmax cross-entropy is a convex problem — no local minima
  • Linear probes reveal what is directly accessible in the representation
  • More complex probes risk learning features not actually used by the model

Key Finding: Monotonic Improvement

ResNet-50 Results (ImageNet)

Layer Topology Validation Error
input (224,224,3) 0.99
add_1 (28,28,256) 0.94
add_4 (28,28,512) 0.87
add_8 (14,14,1024) 0.77
add_13 (14,14,1024) 0.51
add_16 (7,7,2048) 0.31

Error decreases almost perfectly monotonically as we go deeper.

Why is this surprising?

  • The Data Processing Inequality says information can only decrease through layers: \(I(X;Z) \leq I(X;Y)\)
  • Yet linear separability increases
  • This means networks are not about preserving information — they are about distilling computationally useful representations

Lesson: Neural networks transform data into representations that are increasingly easy for simple classifiers to use, even though total information content may decrease.

Diagnosing Model Problems with Probes

Practical use case: finding “dead” segments

Alain & Bengio tested a pathologically deep model (128 layers) on MNIST with a long skip connection bypassing the first half.

What probes revealed:

  • The model trained successfully (loss went down)
  • But probes showed that the first 64 layers were completely unused — no useful signal passed through them
  • The skip connection created a “dead segment”

Takeaway: Loss minimization alone can hide architectural problems. Probes give us internal visibility into what each part of the model is actually doing.

Layers 0–63: dead (no useful signal)

Layers 64–127: doing all the work

Part 2: Promises & Pitfalls

Belinkov (2022): A Critical Review

“Probing Classifiers: Promises, Shortcomings, and Advances”

The probing framework involves many components — choices matter:

Component Description
\(f\) Original model (e.g., BERT, GPT)
\(D_O\) Original training dataset
\(g\) Probing classifier
\(D_P\) Probing dataset with labels \(z\)
\(\text{PERF}(g, f, D_O, D_P)\) Probing performance

Key insight: Probing performance depends on ALL of these components and their interactions — not just on \(f\).

What good probing performance might mean:

  • ✅ Property \(z\) is encoded in representations
  • ✅ Information is extractable from representations
  • ⚠️ The model uses this information (not guaranteed!)
  • ❌ The model has learned the property (could be incidental)

Shortcoming 1: Comparisons & Controls

Problem: If a probe gets 87.8% accuracy, is that high? Compared to what?

Baselines (\(\underline{f}\)):

  • Majority class baseline
  • Random embeddings baseline
  • Even random features can encode surprisingly much information!

Skylines (\(\bar{f}\)):

  • Human performance
  • State-of-the-art dedicated models
  • Dedicated model trained on \(x \rightarrow z\) directly

Hewitt & Liang (2019) — Control Tasks:

Probe may be memorizing rather than extracting real information.

Solution: Randomize labels → create \(D_{P,\text{Rand}}\)

\[\text{Selectivity} = \text{PERF}(g, f, D_P) - \text{PERF}(g, f, D_{P,\text{Rand}})\]

Key finding:

  • Linear probes → high selectivity ✅
  • Nonlinear probes → low selectivity (memorization!) ❌

This is one reason the field prefers linear probes.

Shortcoming 2: Correlation ≠ Causation

The fundamental limitation:

A probe tells us information exists in a representation — but does the model actually use it?

Evidence of disconnect:

  • Vanmassenhove et al. (2017): mismatch between probe performance and model performance
  • Elazar et al. (2021): removing probed features often does NOT hurt original task performance

Causal approaches (interventions):

  • Giulianelli et al. (2018): modify representations using probe gradients → measure effect on original task
  • Feder et al. (2021): adversarial removal of features \(z\) while controlling for confounders

Bottom line: Probes establish correlation. Causal claims require interventions.

Shortcoming 3: Which Classifier?

Approach Pros Cons
Linear probe Convex, high selectivity, reveals what’s “directly readable” May underestimate encoded information
MLP probe Can find more complex patterns Low selectivity, risks memorization
MDL-based (Voita & Titov 2020) Measures both performance AND complexity More complex to implement
Parameter-free (attention-based) Avoids learning new parameters entirely Limited to what attention reveals

Tip

Practical recommendation: Use linear probes as the default. If you need to argue a property is truly represented, supplement with control tasks and selectivity metrics.

Part 3: Modern Applications

Gurnee & Tegmark (2024): Space & Time in LLMs

“Language Models Represent Space and Time” (ICLR 2024)

Key question: Do LLMs just learn superficial statistics, or do they build coherent world models?

Approach: Use linear probes on Llama-2 (7B, 13B, 70B) to predict:

  • Spatial coordinates (latitude/longitude) of places
  • Temporal coordinates (dates) of events

Six datasets spanning multiple scales:

Spatial Temporal
World places (39K) Historical figures (37K)
US places (29K) Art & entertainment (31K)
NYC places (19K) News headlines (28K)

Probe formulation:

Linear ridge regression:

\[\hat{W} = \arg\min_W \|Y - AW\|^2_2 + \lambda\|W\|^2_2\]

where \(A \in \mathbb{R}^{n \times d_\text{model}}\) are activations and \(Y\) contains spatial or temporal coordinates.

Closed-form solution:

\[\hat{W} = (A^T A + \lambda I)^{-1} A^T Y\]

Results: LLMs Build Maps of the World

Main findings from Figure 2 of the paper:

  • Spatial & temporal features are linearly recoverable from LLM activations
  • Representation quality increases through the first half of layers, then plateaus
  • Larger models produce better representations (70B >> 13B >> 7B)
  • Gap between Llama-2 and Pythia likely due to training corpus size (2T vs 300B tokens)

Nonlinear probes (MLP) show minimal improvement over linear probes → evidence that representations are truly linear.

R² comparison (at 60% layer depth):

Dataset Linear MLP
World 0.911 0.926
USA 0.864 0.869
NYC 0.359 0.312
Historical 0.835 0.839
Entertainment 0.885 0.884
Headlines 0.746 0.739

(Llama-2-70B results)

The MLP barely improves — and sometimes does worse — strong evidence for the linear representation hypothesis.

Robustness: Generalization & Prompting

Prompting sensitivity (§3.3):

  • Explicit prompts (“What is the latitude of…”) make little difference
  • Random distractor tokens significantly degrade performance
  • Capitalizing entity names also hurts (interferes with tokenization)
  • Probing on a period token after headlines improves results

This suggests spatial/temporal features are encoded as part of entity processing, not triggered by specific questions.

Block holdout generalization (§4.1):

Hold out entire countries/states/decades and test if probes generalize:

Dataset Nominal Error Held-Out Error
World 0.071 0.164
USA 0.121 0.188
Historical 0.115 0.119
Entertainment 0.105 0.122

(Llama-2-70B, proximity error)

Performance degrades but remains well above random (0.5), suggesting the model truly represents spatial structure, not just category membership.

Space & Time Neurons (§5)

Finding individual neurons that encode coordinates:

  1. Train linear probe → get probe direction vector
  2. Search for individual neurons whose weights have high cosine similarity with the probe direction
  3. Project activations onto these neuron weights

Results: Individual neurons are highly predictive of spatial/temporal coordinates — they respond to space/time across all entity types.

Causal verification:

  • Ablating these neurons degrades the model’s spatial/temporal reasoning
  • These neurons emerged purely from next-token prediction — no explicit supervision

Significance:

  • Linear probes = upper bound on how well the model represents these features
  • Individual neurons = lower bound (features are typically distributed in superposition)
  • The fact that single neurons are this predictive is strong evidence the model truly uses these features

Note

This connects to the broader linear representation hypothesis: features in neural networks are represented as directions in activation space.

Beyond NLP: Probes in Vision (DINOv3)

The probing idea extends to vision models:

DINOv3 (Siméoni et al., 2025) — a self-supervised vision foundation model — is evaluated extensively with linear probes and k-NN classifiers on frozen features.

How probes are used:

  • Linear probing on ImageNet: Train a linear classifier on frozen DINOv3 features → measures quality of learned representations
  • Dense prediction probing: Evaluate frozen backbone features for segmentation, depth estimation, etc.

Key results: DINOv3 achieves state-of-the-art linear probe accuracy across many benchmarks — validating that SSL learns powerful, general features.

Why this matters for our topic:

  • Alain & Bengio (2016) originally applied probes to vision models (ResNet, Inception)
  • DINOv3 shows probing is now a standard evaluation protocol for foundation models
  • The same principle applies: if a linear probe on frozen features can do well → the model has learned useful representations

Probes as evaluation vs. analysis:

Use Goal
Analysis Understand what each layer learns
Evaluation Benchmark representation quality
Generalization Transfer to new tasks cheaply

Related Project: Probes for LLM Generalization

Project: “Using Simple Statistical Probes to Generalize LLMs”

Idea: Use the Alain & Bengio approach to repurpose pre-trained LLMs for new tasks they weren’t trained on.

How it works:

  1. Take a pre-trained LLM from Hugging Face
  2. Pass data through the model and extract middle-layer activations
  3. Train a simple linear model (logistic regression or ridge regression) on these activations for your new task
  4. Evaluate on held-out test data

Why this is powerful:

  • No fine-tuning needed — the LLM stays frozen
  • Extremely lightweight — train a linear model in seconds
  • The middle layers contain richer representations than the final layer for many tasks

Supported by the papers we’ve discussed:

  • Alain & Bengio: Middle layers of deep nets encode increasingly useful representations
  • Gurnee & Tegmark: LLMs learn linear representations of real-world concepts (space, time) that probes can extract
  • Belinkov: Probing reveals rich linguistic information at various layers

Potential experiments:

  • Probe for sentiment, topic, factual attributes
  • Compare probe accuracy across layers
  • Compare different pre-trained LLMs
  • Test on domain-specific data (medical, legal, etc.)

Summary

Key takeaways:

  1. Probes are simple classifiers that measure what information is encoded in neural network representations — without modifying the model

  2. Linear separability increases monotonically through network depth (Alain & Bengio, 2016)

  3. The probing methodology has important limitations: need for proper controls, risk of memorization, correlation ≠ causation (Belinkov, 2022)

  4. Modern LLMs learn linear representations of space and time that can be extracted with simple ridge regression probes (Gurnee & Tegmark, 2024)

The bigger picture:

  • Probes are now a standard tool for both analysis and evaluation in NLP and computer vision
  • The linear representation hypothesis — that neural networks encode features as directions in activation space — has strong empirical support
  • Probing provides a practical, lightweight way to repurpose pre-trained models for new tasks

Tip

Project connection: The probing framework directly enables using frozen LLMs as feature extractors for new classification and regression tasks — the core idea of Project 1.
